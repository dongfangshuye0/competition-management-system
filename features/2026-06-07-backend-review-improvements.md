# 后端复查改进记录

## 目标

重新检查当前高校竞赛管理系统实现，修复安全、业务校验、异常响应和项目文档方面的低风险缺口。

## 修改范围

- 后端统一异常处理
- CORS 与 JWT 配置
- 用户登录、密码修改、用户删除校验
- 报名审核和作品提交业务规则
- Spring Boot 回归测试
- 架构文档补充

## 核心实现

- 业务异常根据业务 code 返回对应 HTTP 状态，系统异常只返回通用提示并写入日志。
- CORS 默认改为本地开发来源白名单，并支持通过 `APP_CORS_ALLOWED_ORIGINS` 配置。
- JWT secret、过期时间和上传目录支持环境变量覆盖。
- 登录失败统一返回“账号或密码错误”，避免暴露账号是否存在。
- 修改密码和管理员创建/重置密码时统一校验最小长度。
- 删除用户前检查是否已关联报名或评审记录，避免产生悬空业务数据。
- 报名已提交作品后禁止再变更审核结果。
- 作品提交必须等竞赛进入 `submission_open` 状态，报名期内不再展示可提交状态。
- 增加针对提交窗口、关联用户删除和短密码的回归测试。

## 影响范围

- 前端仍可读取统一响应体中的 `code` 与 `message`，但 HTTP 状态码现在与业务状态保持一致。
- 使用独立前端开发服务器访问接口时，需要来源在 `APP_CORS_ALLOWED_ORIGINS` 中。
- 学生在报名期内即使报名已通过，也需要等报名截止后进入作品提交期才能上传作品。
- 已有关联报名或评分的用户不能直接删除。

## 验证方式

- 在 `backend/competition` 执行 `.\mvnw.cmd test`。
- 验证结果：4 个测试全部通过。

## 已知限制

- 数据库持久化已在后续改进中接入，主业务数据保存到 MySQL。
- DeepSeek 相关能力仍为本地规则模拟。
- 文件上传仅做基础文件名清理，尚未按文件类型、病毒扫描或配额做生产级限制。
