# 工作台统计卡片跳转

## 目标

让工作台统计卡片中已有对应页面的项目可以点击进入详情页面，避免用户看到数字入口却无法进入。

## 修改范围

- `frontend/src/views/DashboardView.vue`
- `frontend/src/styles.css`

## 核心实现

- 按用户角色维护统计字段到页面路由的映射。
- 管理员统计可进入用户、竞赛、报名、作品、成绩和通知页面。
- 教师统计可进入报名审核、作品评审和通知页面。
- 学生统计可进入竞赛列表、我的报名、我的作品和通知页面。
- 没有对应页面的统计项保持不可点击，避免制造无效入口。
- 可点击卡片使用原生链接，增加 hover、键盘焦点和“查看”提示。

## 影响范围

- 影响管理员、教师、学生工作台统计卡片交互。
- 不影响统计接口返回结构和后台侧边栏导航。

## 验证方式

- 运行前端构建检查类型和打包。
- 在浏览器打开学生工作台，点击“我的报名”等卡片，确认可以跳转到对应页面。
- 如果浏览器评论标注层覆盖卡片，自动化点击会被标注层拦截；可通过检查卡片 `href` 和直接访问目标路由确认应用行为。

## 已知限制

- “学生总数”等暂无独立详情页的统计项仍不提供点击行为。
