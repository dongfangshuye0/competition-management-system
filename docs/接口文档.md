# 高校竞赛管理系统接口文档

## 1. 接口规范

### 1.1 基础信息
- **Base URL**：`http://localhost:8080`
- **数据格式**：JSON
- **统一响应格式**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

| code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证/登录已过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 1.2 认证方式
所有需登录的接口在请求头中携带 JWT Token：

```
Authorization: Bearer <token>
```

## 2. 公共接口（无需登录）

### 2.1 公开竞赛列表

```
GET /api/public/competitions
```

**响应示例：**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": [
    {
      "id": 1,
      "name": "全国大学生软件设计大赛",
      "description": "面向软件工程、计算机与人工智能相关专业...",
      "organizer": "信息工程学院",
      "level": "国家级",
      "registrationStartTime": "2026-06-02T10:00:00",
      "registrationEndTime": "2026-06-14T10:00:00",
      "submissionEndTime": "2026-06-27T10:00:00",
      "status": "registration_open",
      "statusLabel": "报名中",
      "rankingPublished": false,
      "registrationCount": 2
    }
  ]
}
```

### 2.2 公开竞赛详情

```
GET /api/public/competitions/{id}
```

| 参数 | 位置 | 类型 | 说明 |
|------|------|------|------|
| id | Path | Long | 竞赛ID |

**响应**：竞赛详情（同列表项）+ 若已公布成绩则含 ranking 数组

### 2.3 公开排名

```
GET /api/public/competitions/{id}/ranking
```

| 参数 | 位置 | 类型 | 说明 |
|------|------|------|------|
| id | Path | Long | 竞赛ID |

**响应示例：**
```json
{
  "code": 200,
  "data": [
    {
      "rank": 1,
      "registrationId": 1,
      "studentName": "陈同学",
      "teamName": "智创队",
      "title": "智慧竞赛管理平台",
      "finalScore": 88,
      "finalComment": "功能闭环较完整..."
    }
  ]
}
```

> 注：成绩未公布时返回 403。

## 3. 认证模块

### 3.1 登录

```
POST /api/auth/login
Content-Type: application/json
```

**请求体：**
```json
{
  "username": "teacher01",
  "password": "teacher123"
}
```

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "user": {
      "id": 2,
      "username": "teacher01",
      "name": "王老师",
      "role": "teacher",
      "major": "计算机科学与技术",
      "className": null,
      "phone": "13900000001",
      "email": "teacher01@campus.local",
      "createTime": "2026-06-07T10:00:00",
      "updateTime": "2026-06-07T10:00:00"
    }
  }
}
```

### 3.2 学生注册

```
POST /api/auth/register
Content-Type: application/json
```

**请求体：**
```json
{
  "username": "20240003",
  "password": "mypassword",
  "name": "赵同学",
  "major": "计算机科学与技术",
  "className": "计科1班",
  "phone": "13600000000",
  "email": "zhao@campus.local"
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| username | 是 | 学号 |
| password | 是 | 密码（≥6位） |
| name | 是 | 真实姓名 |
| major | 否 | 专业 |
| className | 否 | 班级 |
| phone | 否 | 手机号 |
| email | 否 | 邮箱 |

**响应**：`UserView`（同登录返回的 user 对象）

## 4. 用户模块

**权限**：所有接口需登录（特殊标注除外）

### 4.1 获取当前用户信息

```
GET /api/users/me
```

**响应**：`UserView`

### 4.2 修改个人资料

```
PUT /api/users/me
Content-Type: application/json
```

**请求体：**
```json
{
  "name": "新名字",
  "major": "软件工程",
  "className": "软件1班",
  "phone": "13800001111",
  "email": "new@campus.local",
  "password": "newpassword"
}
```

> 所有字段可选。password 非空则更新。

**响应**：`UserView`

### 4.3 获取所有用户（Admin）

```
GET /api/users
```

**权限**：Admin

**响应**：`List<UserView>`

### 4.4 获取教师列表

```
GET /api/users/teachers
```

**响应**：`List<UserView>`（仅 role=teacher 的用户）

### 4.5 创建用户（Admin）

```
POST /api/users
Content-Type: application/json
```

**请求体：**
```json
{
  "username": "teacher03",
  "password": "teacher123",
  "name": "张老师",
  "role": "teacher",
  "major": "人工智能",
  "className": null,
  "phone": "13900000003",
  "email": "teacher03@campus.local"
}
```

**权限**：Admin

**响应**：`UserView`

### 4.6 更新用户（Admin）

```
PUT /api/users/{id}
Content-Type: application/json
```

**权限**：Admin

**请求体**：同上（id 非空时为更新，可省略 password 以保留原密码）

### 4.7 删除用户（Admin）

```
DELETE /api/users/{id}
```

**权限**：Admin

**约束**：不能删除自己；不能删除最后一个管理员

## 5. 竞赛模块

### 5.1 获取推荐竞赛（学生）

```
GET /api/competitions/recommended
```

**权限**：学生

**响应**：`List<CompetitionView>`（额外含 recommendationScore 和 recommendationReason）

### 5.2 获取竞赛排名（登录）

```
GET /api/competitions/{id}/ranking
```

**权限**：登录用户

### 5.3 创建竞赛（Admin）

```
POST /api/competitions
Content-Type: application/json
```

**权限**：Admin

**请求体：**
```json
{
  "name": "全国大学生AI创新大赛",
  "description": "面向全校的人工智能创新项目竞赛",
  "organizer": "人工智能学院",
  "level": "国家级",
  "registrationStartTime": "2026-07-01T00:00:00",
  "registrationEndTime": "2026-07-15T00:00:00",
  "submissionEndTime": "2026-08-01T00:00:00"
}
```

**约束**：报名开始时间 ≤ 报名截止时间 ≤ 作品提交截止时间

**响应**：`CompetitionView`

### 5.4 更新竞赛（Admin）

```
PUT /api/competitions/{id}
Content-Type: application/json
```

**权限**：Admin

### 5.5 删除竞赛（Admin）

```
DELETE /api/competitions/{id}
```

**权限**：Admin

**约束**：已有报名记录的竞赛不能删除

### 5.6 公布成绩（Admin）

```
POST /api/competitions/{id}/publish-ranking
```

**权限**：Admin

**说明**：公布后学生和教师可查看排名

## 6. 报名模块

### 6.1 报名竞赛（学生）

```
POST /api/registrations
Content-Type: multipart/form-data
```

**权限**：学生

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| competitionId | Long | 是 | 竞赛ID |
| teacherId | Long | 是 | 指导教师ID |
| teamName | String | 否 | 队伍名称 |
| teamMembers | String | 否 | 成员信息 |
| applicationForm | File | 否 | 报名表文件上传 |

**约束**：
- 同一竞赛不可重复报名
- 必须在报名时间窗口内

**响应**：`RegistrationView`

### 6.2 我的报名（学生）

```
GET /api/registrations/my
```

**权限**：学生

**响应**：`List<RegistrationView>`

**RegistrationView 字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 报名ID |
| competitionId | Long | 竞赛ID |
| competitionName | String | 竞赛名称 |
| competitionLevel | String | 竞赛级别 |
| studentId | Long | 学生ID |
| studentName | String | 学生姓名 |
| studentUsername | String | 学生学号 |
| teacherId | Long | 教师ID |
| teacherName | String | 教师姓名 |
| teamName | String | 队伍名称 |
| teamMembers | String | 成员信息 |
| applicationForm | String | 报名表文件路径 |
| status | String | pending/approved/rejected |
| statusLabel | String | 待审核/已通过/未通过 |
| comment | String | 审核意见 |
| createTime | DateTime | 创建时间 |
| workSubmitted | Boolean | 是否已提交作品 |
| canSubmitWork | Boolean | 是否可提交作品 |

### 6.3 教师审核列表

```
GET /api/registrations/teacher
```

**权限**：教师/Admin

**说明**：教师查看自己指导的报名；Admin 查看所有

### 6.4 全部报名（Admin）

```
GET /api/registrations
```

**权限**：Admin

### 6.5 审核报名

```
PUT /api/registrations/{id}/review
Content-Type: application/json
```

**权限**：教师/Admin

**请求体：**
```json
{
  "status": "approved",
  "comment": "材料完整，同意参赛"
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| status | 是 | approved（通过）或 rejected（驳回） |
| comment | 否 | 审核意见 |

## 7. 作品模块

### 7.1 提交作品（学生）

```
POST /api/works
Content-Type: multipart/form-data
```

**权限**：学生

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| registrationId | Long | 是 | 报名ID（需已通过审核） |
| title | String | 是 | 作品标题 |
| description | String | 否 | 作品描述 |
| file | File | 否 | 作品文件上传 |

**说明**：
- 提交后自动触发 **DeepSeek AI 初评**（生成 aiScore 和 aiComment）
- 同一报名只能提交一次作品

**响应**：`WorkView`

### 7.2 我的作品（学生）

```
GET /api/works/my
```

**权限**：学生

**响应**：`List<WorkView>`

**WorkView 字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 作品ID |
| registrationId | Long | 报名ID |
| competitionId | Long | 竞赛ID |
| competitionName | String | 竞赛名称 |
| studentName | String | 学生姓名 |
| studentUsername | String | 学生学号 |
| guideTeacherName | String | 指导教师姓名 |
| reviewTeacherId | Long | 评审教师ID |
| reviewTeacherName | String | 评审教师姓名 |
| title | String | 作品标题 |
| description | String | 作品描述 |
| filePath | String | 作品文件路径 |
| submitTime | DateTime | 提交时间 |
| aiScore | Integer | AI初评分 |
| aiComment | String | AI初评评语 |
| finalScore | Integer | 最终得分（AI分或教师平均分） |
| finalComment | String | 最终评语 |
| scores | Array | 教师评分列表 |

### 7.3 教师评审列表

```
GET /api/works/teacher
```

**权限**：教师/Admin

**说明**：教师查看自己负责评审的作品；Admin 查看所有

### 7.4 全部作品（Admin）

```
GET /api/works
```

**权限**：Admin

### 7.5 分配评审教师（Admin）

```
PUT /api/works/{id}/assign-reviewer
Content-Type: application/json
```

**权限**：Admin

**请求体：**
```json
{
  "reviewerId": 2
}
```

### 7.6 作品评分

```
POST /api/works/{id}/score
Content-Type: application/json
```

**权限**：教师/Admin

**请求体：**
```json
{
  "score": 88,
  "comment": "项目完成度高，建议补充测试数据"
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| score | 是 | 0-100 整数 |
| comment | 否 | 评语 |

## 8. 通知模块

### 8.1 我的通知

```
GET /api/notifications
```

**权限**：登录

**响应**：`List<NotificationView>`

**NotificationView 字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 通知ID |
| title | String | 通知标题 |
| content | String | 通知内容 |
| read | Boolean | 是否已读 |
| createTime | DateTime | 创建时间 |

### 8.2 标记已读

```
PUT /api/notifications/{id}/read
```

**权限**：登录

**说明**：只能标记自己的通知

### 8.3 发布通知（Admin）

```
POST /api/notifications
Content-Type: application/json
```

**权限**：Admin

**请求体：**
```json
{
  "title": "竞赛报名提醒",
  "content": "全国大学生软件设计大赛报名即将截止，请抓紧时间。",
  "userId": null,
  "broadcast": true
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| title | 是 | 通知标题 |
| content | 是 | 通知内容 |
| userId | 否 | 指定用户ID（broadcast=true时忽略） |
| broadcast | 否 | true=全体发送，false=指定用户 |

**响应：**
```json
{
  "code": 200,
  "data": {
    "broadcast": true,
    "count": 5
  }
}
```
