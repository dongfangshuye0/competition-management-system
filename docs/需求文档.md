# 高校竞赛管理系统需求文档

## 1. 项目背景
为了解决高校竞赛管理中存在的报名流程繁琐、信息通知不及时、作品管理混乱等问题，开发一个基于 Spring Boot + Vue3 前后端分离的高校竞赛管理系统，并集成 DeepSeek 大模型提供智能化服务。

## 2. 技术栈

### 2.1 后端（Spring Boot）

| 层级 | 技术 | 版本 | 用途 |
|------|------|------|------|
| 语言 | Java | 17 | 编程语言 |
| 框架 | Spring Boot | 3.2.6 | 应用框架 |
| ORM | MyBatis-Plus | 3.5.10 | ORM / MySQL 持久化 |
| 安全 | Spring Security Crypto / JWT | 6.2.4 / 0.11.5 | BCrypt 密码加密 + Token 认证 |
| 工具 | Lombok / Hutool | - | 代码简化 / 工具类库 |
| 构建 | Maven | - | 项目构建 |

### 2.2 前端（Vue3）

| 层级 | 技术 | 版本 | 用途 |
|------|------|------|------|
| 语言 | TypeScript | 5.x | 编程语言 |
| 框架 | Vue | 3.x | 前端框架 |
| 构建 | Vite | 5.x | 构建工具 / 开发服务器 |
| UI 库 | Element Plus | 2.x | 组件库 |
| 路由 | Vue Router | 4.x | 前端路由 |
| 状态管理 | Pinia | 2.x | 全局状态管理 |
| HTTP 请求 | Axios | 1.x | 后端 API 调用 |
| 运行时 | Node.js | ≥18 | 开发环境运行 |

> 注：系统后端连接 MySQL `competition_db`，所有用户、竞赛、报名、作品、评分和通知数据都会持久化保存。`sql/init.sql` 用于初始化或升级数据库结构。前端通过 Axios 请求后端 RESTful API，开发模式下通过 Vite 代理转发请求解决跨域。

## 3. 用户角色

| 角色 | 标识 | 说明 |

## 5. 核心功能清单
### 5.1 学生端功能
- 注册、登录、修改个人信息（姓名、学号、专业、班级、手机号、邮箱）
- 浏览竞赛列表、查看竞赛详情（含公开排名）
- 在线报名竞赛（上传报名表、选择指导教师、填写队伍信息）
- 提交竞赛作品（上传文件、填写作品标题和描述）
- 查看报名状态（待审核/已通过/未通过）、作品评审结果（AI初评+教师评分）
- 接收系统通知
- 智能答疑助手（DeepSeek）
- 竞赛智能推荐（DeepSeek）

### 5.2 教师端功能
- 查看所指导学生的报名信息
- 审核学生的竞赛报名申请（通过/驳回）
- 查看被分配评审的学生作品
- 给作品打分（0-100）和写评语
- 接收系统通知

### 5.3 管理员端功能
- 用户管理（增删改查学生、教师账号）
- 竞赛管理（发布、修改、删除竞赛信息）
- 报名管理（查看所有报名信息）
- 作品管理（查看所有作品、分配评审教师）
- 成绩管理（公布成绩、生成排名）
- 通知管理（发布系统通知，可指定用户或全体广播）

## 6. 竞赛状态说明

系统根据当前时间自动计算竞赛状态：

| 状态标识 | 中文标签 | 条件 |
|----------|----------|------|
| `upcoming` | 即将开始 | 当前时间 < 报名开始时间 |
| `registration_open` | 报名中 | 报名开始时间 ≤ 当前时间 ≤ 报名截止时间 |
| `submission_open` | 作品提交中 | 报名截止时间 < 当前时间 ≤ 作品提交截止时间 |
| `ended` | 已结束 | 当前时间 > 作品提交截止时间 |

### 7.3 开发模式说明

| 模块 | 端口 | 启动命令 |
|------|------|----------|
| 后端 (Spring Boot) | 8080 | `mvn spring-boot:run` |
| 前端 (Vite Dev Server) | 5173 | `npm run dev` |
| 数据库 (MySQL) | 3306 | 需提前启动 |

> 前端开发时通过 Vite 代理 `/api` 到 `http://localhost:8080`，无需额外跨域配置。生产环境将前端构建产物部署到 Nginx 或对象存储，反向代理到后端。

## 9. DeepSeek 亮点功能
1. **智能答疑助手**：解答学生关于竞赛报名、规则、提交等常见问题，结合系统当前数据给出具体建议
2. **竞赛作品智能初评**：对学生提交的论文/报告类作品进行自动评分和评语生成

## 11. 非功能需求
- 系统响应时间：页面加载时间不超过3秒
- 兼容性：支持 Chrome、Edge、Firefox 浏览器
- 安全性：用户密码 BCrypt 加密存储，接口需要 JWT 身份认证
- 跨域：后端配置 CORS 允许前端开发服务器（localhost:5173）访问
- 部署方式：后端打包为 JAR 运行，前端通过 Nginx 或 CDN 部署静态文件
