# 高校竞赛管理系统数据库设计

## 1. 数据库概览

- **数据库名**：`competition_db`
- **字符集**：`utf8mb4` / `utf8mb4_unicode_ci`
- **存储引擎**：InnoDB
- **表数量**：6 张

## 2. E-R 关系

| 主表 | 从表 | 关系 | 外键 |
|------|------|------|------|
| `user`（学生） | `registration` | 一对多 | `registration.user_id → user.id` |
| `user`（教师） | `registration` | 一对多 | `registration.teacher_id → user.id` |
| `competition` | `registration` | 一对多 | `registration.competition_id → competition.id` |
| `registration` | `work` | 一对一 | `work.registration_id → registration.id` |
| `work` | `score` | 一对多 | `score.work_id → work.id` |
| `user`（教师） | `score` | 一对多 | `score.teacher_id → user.id` |
| `user` | `notification` | 一对多 | `notification.user_id → user.id` |

## 3. 表结构详细说明

### 3.1 user（用户表）

存储学生、教师、管理员账号信息。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `bigint` | PK, AUTO_INCREMENT | 用户ID |
| `username` | `varchar(100)` | NOT NULL, UNIQUE | 用户名/学号/工号 |
| `password` | `varchar(200)` | NOT NULL | BCrypt 加密存储 |
| `name` | `varchar(50)` | NOT NULL | 真实姓名 |
| `role` | `varchar(20)` | NOT NULL, DEFAULT 'student' | 角色：student / teacher / admin |
| `major` | `varchar(100)` | DEFAULT NULL | 专业 |
| `class_name` | `varchar(100)` | DEFAULT NULL | 班级（实体类中字段名为 className） |
| `phone` | `varchar(20)` | DEFAULT NULL | 手机号 |
| `email` | `varchar(100)` | DEFAULT NULL | 邮箱 |
| `create_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `update_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| `is_deleted` | `tinyint(1)` | DEFAULT '0' | 逻辑删除：0-否，1-是 |

**索引**：`PRIMARY KEY (id)`，`UNIQUE KEY uk_username (username)`

---

### 3.2 competition（竞赛表）

存储竞赛的基本信息、时间窗口和发布状态。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `bigint` | PK, AUTO_INCREMENT | 竞赛ID |
| `name` | `varchar(200)` | NOT NULL | 竞赛名称 |
| `description` | `text` | DEFAULT NULL | 竞赛描述 |
| `organizer` | `varchar(200)` | DEFAULT NULL | 主办方 |
| `level` | `varchar(20)` | DEFAULT NULL | 竞赛级别：校级 / 省级 / 国家级 |
| `registration_start_time` | `datetime` | NOT NULL | 报名开始时间 |
| `registration_end_time` | `datetime` | NOT NULL | 报名截止时间 |
| `submission_end_time` | `datetime` | NOT NULL | 作品提交截止时间 |
| `ranking_published` | `tinyint(1)` | DEFAULT '0' | 成绩是否公布：0-否，1-是 |
| `create_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `update_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| `is_deleted` | `tinyint(1)` | DEFAULT '0' | 逻辑删除：0-否，1-是 |

**索引**：`PRIMARY KEY (id)`

> 注：竞赛状态由后端根据当前时间动态计算，不落库保存。

---

### 3.3 registration（报名表）

存储学生的竞赛报名记录。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `bigint` | PK, AUTO_INCREMENT | 报名ID |
| `user_id` | `bigint` | NOT NULL | 学生ID → user.id |
| `competition_id` | `bigint` | NOT NULL | 竞赛ID → competition.id |
| `teacher_id` | `bigint` | DEFAULT NULL | 指导教师ID → user.id |
| `team_name` | `varchar(100)` | DEFAULT NULL | 队伍名称 |
| `team_members` | `text` | DEFAULT NULL | 团队成员信息 |
| `application_form` | `varchar(500)` | DEFAULT NULL | 报名表文件路径 |
| `status` | `varchar(20)` | DEFAULT 'pending' | 状态：pending / approved / rejected |
| `comment` | `varchar(500)` | DEFAULT NULL | 审核意见 |
| `create_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `update_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| `is_deleted` | `tinyint(1)` | DEFAULT '0' | 逻辑删除：0-否，1-是 |

**索引**：`PRIMARY KEY (id)`，`KEY idx_user_id (user_id)`，`KEY idx_competition_id (competition_id)`，`KEY idx_teacher_id (teacher_id)`

---

### 3.4 work（作品表）

存储学生提交的作品信息，包含 AI 初评结果。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `bigint` | PK, AUTO_INCREMENT | 作品ID |
| `registration_id` | `bigint` | NOT NULL, UNIQUE | 报名ID → registration.id |
| `title` | `varchar(200)` | NOT NULL | 作品标题 |
| `description` | `text` | DEFAULT NULL | 作品描述 |
| `file_path` | `varchar(500)` | NOT NULL | 作品文件路径 |
| `review_teacher_id` | `bigint` | DEFAULT NULL | 评审教师ID |
| `ai_score` | `int` | DEFAULT NULL | AI初评分 |
| `ai_comment` | `text` | DEFAULT NULL | AI初评评语 |
| `submit_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP | 提交时间 |
| `create_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `update_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| `is_deleted` | `tinyint(1)` | DEFAULT '0' | 逻辑删除：0-否，1-是 |

**索引**：`PRIMARY KEY (id)`，`UNIQUE KEY uk_registration_id (registration_id)`

---

### 3.5 score（成绩表）

存储教师对作品的评分记录。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `bigint` | PK, AUTO_INCREMENT | 成绩ID |
| `work_id` | `bigint` | NOT NULL | 作品ID → work.id |
| `teacher_id` | `bigint` | NOT NULL | 评审教师ID → user.id |
| `score` | `int` | NOT NULL | 评分（0-100） |
| `comment` | `text` | DEFAULT NULL | 评语 |
| `ai_score` | `int` | DEFAULT NULL | AI初评分 |
| `ai_comment` | `text` | DEFAULT NULL | AI初评评语 |
| `create_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `update_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| `is_deleted` | `tinyint(1)` | DEFAULT '0' | 逻辑删除：0-否，1-是 |

**索引**：`PRIMARY KEY (id)`，`KEY idx_work_id (work_id)`，`KEY idx_teacher_id (teacher_id)`

> 注：实体类中 `aiScore` 和 `aiComment` 实际放在 `Work` 实体中，SQL 脚本中 `score` 表含有这两个字段。当前系统在 `Work` 实体中保存 AI 评审结果，`Score` 实体保存教师评分。

---

### 3.6 notification（通知表）

存储系统通知消息。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `bigint` | PK, AUTO_INCREMENT | 通知ID |
| `title` | `varchar(200)` | NOT NULL | 通知标题 |
| `content` | `text` | NOT NULL | 通知内容 |
| `user_id` | `bigint` | DEFAULT NULL | 接收用户ID → user.id（NULL表示全体用户） |
| `is_read` | `tinyint(1)` | DEFAULT '0' | 是否已读：0-否，1-是 |
| `create_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `update_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| `is_deleted` | `tinyint(1)` | DEFAULT '0' | 逻辑删除：0-否，1-是 |

**索引**：`PRIMARY KEY (id)`，`KEY idx_user_id (user_id)`

## 4. 关系总结图

```
user (学生) ──┐
              ├──→ registration ──→ work ──→ score
user (教师) ──┘       │                        │
                      │                        └── user (评审教师)
                      └── competition

user (任意) ──→ notification
```

## 5. 完整建表 SQL

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS competition_db
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

USE competition_db;

-- 用户表
CREATE TABLE `user` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `username` varchar(100) NOT NULL COMMENT '用户名/学号/工号',
  `password` varchar(200) NOT NULL COMMENT '密码（BCrypt加密）',
  `name` varchar(50) NOT NULL COMMENT '真实姓名',
  `role` varchar(20) NOT NULL DEFAULT 'student' COMMENT '角色：student/teacher/admin',
  `major` varchar(100) DEFAULT NULL COMMENT '专业',
  `class_name` varchar(100) DEFAULT NULL COMMENT '班级',
  `phone` varchar(20) DEFAULT NULL COMMENT '手机号',
  `email` varchar(100) DEFAULT NULL COMMENT '邮箱',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `is_deleted` tinyint(1) DEFAULT '0' COMMENT '逻辑删除：0-否，1-是',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

-- 竞赛表
CREATE TABLE `competition` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '竞赛ID',
  `name` varchar(200) NOT NULL COMMENT '竞赛名称',
  `description` text COMMENT '竞赛描述',
  `organizer` varchar(200) DEFAULT NULL COMMENT '主办方',
  `level` varchar(20) DEFAULT NULL COMMENT '竞赛级别：校级/省级/国家级',
  `registration_start_time` datetime NOT NULL COMMENT '报名开始时间',
  `registration_end_time` datetime NOT NULL COMMENT '报名截止时间',
  `submission_end_time` datetime NOT NULL COMMENT '作品提交截止时间',
  `ranking_published` tinyint(1) DEFAULT '0' COMMENT '成绩是否公布：0-否，1-是',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `is_deleted` tinyint(1) DEFAULT '0' COMMENT '逻辑删除：0-否，1-是',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='竞赛表';

-- 报名表
CREATE TABLE `registration` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '报名ID',
  `user_id` bigint NOT NULL COMMENT '学生ID',
  `competition_id` bigint NOT NULL COMMENT '竞赛ID',
  `teacher_id` bigint DEFAULT NULL COMMENT '指导教师ID',
  `team_name` varchar(100) DEFAULT NULL COMMENT '队伍名称',
  `team_members` text COMMENT '团队成员信息',
  `application_form` varchar(500) DEFAULT NULL COMMENT '报名表文件路径',
  `status` varchar(20) DEFAULT 'pending' COMMENT '状态：pending/approved/rejected',
  `comment` varchar(500) DEFAULT NULL COMMENT '审核意见',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `is_deleted` tinyint(1) DEFAULT '0' COMMENT '逻辑删除：0-否，1-是',
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_competition_id` (`competition_id`),
  KEY `idx_teacher_id` (`teacher_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='报名表';

-- 作品表
CREATE TABLE `work` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '作品ID',
  `registration_id` bigint NOT NULL COMMENT '报名ID',
  `title` varchar(200) NOT NULL COMMENT '作品标题',
  `description` text COMMENT '作品描述',
  `file_path` varchar(500) NOT NULL COMMENT '作品文件路径',
  `review_teacher_id` bigint DEFAULT NULL COMMENT '评审教师ID',
  `ai_score` int DEFAULT NULL COMMENT 'AI初评分',
  `ai_comment` text COMMENT 'AI初评评语',
  `submit_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '提交时间',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `is_deleted` tinyint(1) DEFAULT '0' COMMENT '逻辑删除：0-否，1-是',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_registration_id` (`registration_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='作品表';

-- 成绩表
CREATE TABLE `score` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '成绩ID',
  `work_id` bigint NOT NULL COMMENT '作品ID',
  `teacher_id` bigint NOT NULL COMMENT '评审教师ID',
  `score` int NOT NULL COMMENT '评分（0-100）',
  `comment` text COMMENT '评语',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `is_deleted` tinyint(1) DEFAULT '0' COMMENT '逻辑删除：0-否，1-是',
  PRIMARY KEY (`id`),
  KEY `idx_work_id` (`work_id`),
  KEY `idx_teacher_id` (`teacher_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='成绩表';

-- 通知表
CREATE TABLE `notification` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '通知ID',
  `title` varchar(200) NOT NULL COMMENT '通知标题',
  `content` text NOT NULL COMMENT '通知内容',
  `user_id` bigint DEFAULT NULL COMMENT '接收用户ID（NULL表示全体）',
  `is_read` tinyint(1) DEFAULT '0' COMMENT '是否已读：0-否，1-是',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `is_deleted` tinyint(1) DEFAULT '0' COMMENT '逻辑删除：0-否，1-是',
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='通知表';
```
